# Проверка элементов

Страница, чтобы увидеть, как настройки темы легли на элементы. Переключай светлую/тёмную тему в шапке.

## Заголовок второго уровня

Обычный абзац текста. Вторичный оттенок появляется в подписях и приглушённых местах. Вот **жирный текст** и [ссылка бренд-цветом](#).

### Заголовок третьего уровня

Ещё абзац с `инлайн-кодом` и более длинным предложением, чтобы проверить межстрочный интервал и читаемость основного шрифта.

## Код

```js
// моноширинный шрифт
export default {
  brand: 'accent',
  danger: 'red'
}
```

## Цветные блоки

::: info Инфо
Нейтральный info-блок.
:::

::: tip Подсказка
Бренд-акцент.
:::

::: warning Предупреждение
Тёплый блок-предупреждение.
:::

::: danger Ошибка
Блок ошибки.
:::

## Таблица

| Элемент | Назначение | Пример использования |
|---------|------------|----------------------|
| Заголовки | Задают структуру страницы и иерархию разделов: H1 для названия, H2–H3 для блоков, H4 для мелких подпунктов. | Навигация по странице, оглавление справа, якоря для ссылок. |
| Ссылки | Переходы между страницами и акценты в тексте. Цвет берётся из бренд-токена и одинаков для инлайн-ссылок и пунктов меню. | `[текст](/path)` в абзаце, пункты бокового меню, кнопки-действия. |
| Код | Моноширинный шрифт для инлайн-кода и блоков кода. Используется для команд, имён полей и примеров конфигурации. | `npm run dev`, имена параметров, фрагменты JSON. |
| Блоки-уведомления | Четыре типа: info, tip, warning, danger.<br>Цвет фона и текста задаётся статус-токенами.<br>Warning — тёплый амбер вместо жёлтого. | Подсказки, предупреждения и сообщения об ошибках внутри статьи. |
| Таблицы | Табличные данные с заголовком и чередованием строк. Поддерживают многострочные ячейки — длинный текст переносится автоматически. | Сравнения, списки параметров, матрицы прав и форматов. |
| Кнопки | Основная (brand) и вторичная (alt) — для главных и второстепенных действий. | Кнопки в хедере главной страницы. |
